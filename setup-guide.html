Setup Guide · Faceless Content Toolkit

Getting the Toolkit Running

Everything in faceless-content-toolkit.zip, what each file does, and the shortest path to using it — whether that's opening a page in your browser or wiring it into an agent like Hermes or omp.
Browser tool< 1 minute, free
Agent (MCP) setup~10 minutes
Requires for MCPNode.js, npm

01 What's in the zip

Core files plus the secure local router, with two independent ways to use the same generator. You only need to set up the one(s) you actually want.

faceless-content-toolkit/ ├── faceless_content_generator.html — open in a browser, fill a form, get results on screen ├── secure-provider-server.js — local provider router; keeps API keys out of the browser ├── .env.example — provider presets for OpenCode Zen/Go, Neuralwatt, OpenAI, Anthropic, Codex OAuth ├── index.js — the same generator as an MCP tool; follows your agent's model by default ├── package.json — dependency list and start scripts ├── setup-guide.html — this file └── README.md — condensed reference version of this guide

02 Pick your path

Path A

Just use it now — free

Paste faceless_content_generator.html into a Claude.ai chat and use it as an artifact. No install, no API key, works on Free. Go to §03.

Path B

Let my agent call it

Wire index.js into Hermes, omp, or another MCP client so it's called automatically. Go to §04.

The two paths don't depend on each other — set up one, both, or switch later. Same prompt logic, same output schema either way.

03 Path A — the browser tool

faceless_content_generator.html works two different ways depending on where you open it, and it detects which one automatically:

Where you open itWhat happensCost
Pasted into a Claude.ai chat and turned into an artifactWorks immediately, no router neededFree — covered by any Claude.ai plan, including Free
Downloaded and double-clicked (any browser, no Claude account)Uses the secure local provider router at 127.0.0.1Depends on the provider/key you choose

If you want the free route — run it as a Claude.ai artifact

This is the option that needs no API key and costs nothing beyond your existing Claude.ai usage, on any plan including Free.

  1. Open faceless_content_generator.html in a text editor (or upload the file straight into a Claude.ai chat).
  2. Start a new chat on claude.ai and paste in the full file contents (or attach the file itself), then ask: Turn this into an artifact I can use.
  3. Claude renders it as a live artifact in the side panel — the form, the generate button, all of it, running in that panel.
  4. Fill in the niche/topic, length, tone, and visual style, then click Generate Video Package — no key bar appears, because the artifact detects it's inside Claude and the API call is proxied through your session automatically.
Why this one's free: Anthropic proxies API calls made from inside an artifact using whichever account is currently viewing it — at no extra cost, on any Claude.ai plan. That proxy only exists inside claude.ai/Desktop; it's not available once the file leaves that environment.
Good to know: the artifact lives in that one conversation. If you want it handy again later, keep the chat around, or re-paste the file into a fresh chat any time — the code itself never changes.

If you're running it standalone securely

  1. Copy .env.example to .env.
  2. Add the provider key you want, such as OPENCODE_GO_API_KEY, OPENCODE_ZEN_API_KEY, NEURALWATT_API_KEY, OPENAI_API_KEY, or ANTHROPIC_API_KEY.
  3. Start the router with node secure-provider-server.js or npm run start:router.
  4. Double-click faceless_content_generator.html, pick the provider preset, keep the router URL as http://127.0.0.1:3737, then click Check.
  5. Fill in the niche/topic, length, tone, and visual style, then click Generate Video Package.
Standalone security note: API keys stay in .env on your machine. The browser only sends prompts and provider IDs to the local router; it does not store or transmit API keys.

Recent Generations

Every successful generation is saved automatically and listed under "Recent Generations" — click any entry to reload that full package. Inside Claude it's saved to Claude's built-in storage; standalone it's saved to that browser's local storage, so history doesn't follow you between browsers or devices.

04 Path B — wire it into your agent (MCP)

This runs index.js as a small local process your agent starts on its own and talks to over MCP — no browser tab involved.

1. Unzip somewhere permanent

Terminal
mkdir -p ~/tools
unzip faceless-content-toolkit.zip -d ~/tools/faceless-content-toolkit

2. Install the one dependency

cd ~/tools/faceless-content-toolkit
npm install

3. API key — usually skip this

index.js uses MCP sampling by default: instead of calling the Anthropic API itself, it asks your connected agent to run the completion on whatever model that agent is currently using. That means no API key, no separate billing, and the tool automatically follows your agent if you switch its model later. This works in any client that supports sampling — Claude Desktop and Hermes both do.

A fallback is only needed for clients that do not support sampling. The preferred fallback is the secure provider router: start secure-provider-server.js and set FCT_ROUTER_URL=http://127.0.0.1:3737 in the MCP server env. The older ANTHROPIC_API_KEY fallback still works, but the router is more flexible and keeps all provider presets in one place.

4. (Optional) test it runs before wiring it up

node index.js

It should print [faceless-content-mcp] Server running on stdio. Sampling-first mode; and then sit there waiting — that's correct, it's speaking MCP over stdio and expects a client, not a human typing into it. Ctrl+C to stop.

5. Point your agent's config at it

Hermes and omp use the same idea: a config file listing MCP servers by name, each with a command to run. Start with no env block if your client supports sampling. Add FCT_ROUTER_URL if you want the secure provider router as fallback or direct mode.

Hermes — ~/.hermes/config.yaml
mcp_servers:
  faceless_content:
    command: "node"
    args: ["/Users/you/tools/faceless-content-toolkit/index.js"]
omp — ~/.omp/agent/mcp.json
{
  "mcpServers": {
    "faceless_content": {
      "command": "node",
      "args": ["/Users/you/tools/faceless-content-toolkit/index.js"]
    }
  }
}
Claude Desktop — ~/Library/Application Support/Claude/claude_desktop_config.json (macOS) or %APPDATA%\Claude\claude_desktop_config.json (Windows)
{
  "mcpServers": {
    "faceless_content": {
      "command": "node",
      "args": ["/Users/you/tools/faceless-content-toolkit/index.js"]
    }
  }
}
If your client needs fallback or you want direct provider routing
{
  "mcpServers": {
    "faceless_content": {
      "command": "node",
      "args": ["/Users/you/tools/faceless-content-toolkit/index.js"],
      "env": { "FCT_ROUTER_URL": "http://127.0.0.1:3737" }
    }
  }
}

Same shape works for Hermes' YAML — add FCT_ROUTER_URL: "http://127.0.0.1:3737" under the server entry. Start secure-provider-server.js before running the agent.

Use the absolute path. Write out /Users/you/tools/... in full — some agents don't expand ~ inside JSON/YAML config files, and a wrong path is the most common reason a server fails to register.
Which clients support sampling? Claude Desktop and Hermes both do, so both run this tool key-free by default. If a client doesn't support sampling, every call can fail over to the secure router when FCT_ROUTER_URL is set (and tells you so in its output) — it doesn't just error out.

6. Restart your agent

Hermes, omp, and Claude Desktop all discover MCP servers at startup. Restart the agent (Hermes can also auto-reload on config save, but a restart is the safest bet the first time).

7. Verify it registered

AgentWhere to check
HermesTool list — look for mcp_faceless_content_generate_faceless_content
ompTool list — look for mcp__faceless_content_generate_faceless_content
Claude DesktopHammer icon in the chat input box — click it to see the tool listed

If it's missing, check the agent's logs for a startup error from that server — usually a wrong path, a missing npm install, or a router/provider configuration issue.

05 Using it once it's set up

No special syntax needed either way. In the browser tool, just fill the form. With an agent, describe what you want in plain language — it reads the tool's description and decides on its own when to call it:

Generate a faceless TikTok script about weird space facts, keep it punchy, 30 seconds.
Use the faceless content tool for a Shorts video on budgeting tips for college students.
Give me a storyboard and image prompts for a video about ancient Rome myths, calm and cinematic.

06 What comes back

Both paths return the identical structure — one package covering the whole video:

  • Title and hook line
  • Timestamped script with delivery notes
  • Scene-by-scene storyboard (camera angle, on-screen text, visual description)
  • One image prompt and one video prompt per scene, ready to paste into Midjourney/DALL·E/ChatGPT image gen or Runway/Kling/Sora
  • A caption with hashtags

07 Troubleshooting

Browser tool (Path A)

Key bar keeps appearing. It only disappears when the page detects it's running inside a Claude.ai artifact. A plain downloaded file will always show it — that's expected, not a bug.
"Router/provider issue." Check that node secure-provider-server.js is running and that your selected provider has the required key in .env.
Recent Generations is empty after reopening the file. Standalone history lives in that specific browser's local storage — a different browser, a private/incognito window, or clearing site data will all show an empty list.

Agent / MCP (Path B)

Tool doesn't show up. Confirm the path in your config is absolute and correct, and that npm install completed in that exact folder.
"This MCP client doesn't support sampling" error. Your client can't do MCP sampling, so start the secure router and add FCT_ROUTER_URL=http://127.0.0.1:3737 to the MCP server env.
Auth / API key errors on the fallback path. Double-check your .env file for the selected provider. The browser should never receive the key.
Generation returns malformed output. Rare, but if it happens, just ask your agent to retry — the underlying model occasionally needs a second pass to return clean JSON.
Keeping the key safe. If you did add a fallback key, don't commit the config file to a public repo — it lives there in plain text.
Same prompt, different model source. Path A inside Claude uses Claude. Path A standalone uses your selected secure router provider. Path B follows whatever model your connected agent is currently running via sampling, or falls back to the router if configured. Either way it's the identical prompt and JSON schema, so structure stays consistent — only the exact model generating it may differ.

Troubleshooting: router runs but Generate fails

If http://127.0.0.1:3737 opens, the router is running. Check http://127.0.0.1:3737/health to see which providers are configured.

If the terminal says Configured providers: openai:gpt-5.5 but the UI dropdown is set to Codex OAuth, select Auto or OpenAI API, or enable Codex OAuth first.

codex login
codex login status

# .env
CODEX_OAUTH_ENABLED=true
CODEX_MODEL=gpt-5-codex

npm run start:router
